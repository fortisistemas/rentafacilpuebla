2. Getting Started

Template files are located in the "html" folder of the downloaded archive. Copy all files from that folder to your server.

Note: You need to have hosting with PHP support to use "sending mail" feature.
Note: Some options like sending mail and closing popup gallery are not working when run locally. They will work once you put your site online.

Template uses Bootstrap 3 so it's advised to be familiar with at least Bootstrap grid system: http://getbootstrap.com/css/#grid

You can use one of eight clor skins. You just need to replace link to default blue skin (<link id="skin" rel="stylesheet" type="text/css" href="css/apartment-colors-blue.css" />) in the header with one, located in the css folder. It's also easy to modify it or create your own. Just adjust skin stylesheet.

3. Google Map Support

As template uses many advanced Google Maps features I will discuss them on the beginning.

First of all you need to have Google Maps API included in your page:



<!-- Google Maps -->
<script type="text/javascript" src="http://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY&sensor=false&libraries=places"></script>

You need to obtain your own free API key here: https://developers.google.com/maps/documentation/javascript/

If you don't expect much traffic you can use API without key:



<!-- Google Maps -->
<script type="text/javascript" src="http://maps.googleapis.com/maps/api/js?sensor=false&libraries=places"></script>

Once you're done with API you can use few functions to add maps in your website. All functions should be addend in the page footer, wrapped inside this code:



<!-- google maps initialization -->
<script type="text/javascript">
    google.maps.event.addDomListener(window, 'load', init);
	function init() {						
		..map functions here...
	}
</script>

  • Simple map by latitiude & longitude: 

    mapInit(latitiude, longitiude, map container ID, link to the pin image, user interface (true/false), offset (true/false) - optional);
example:
mapInit(39.6282,-86.1320,"estate-map","images/pin-house.png", true);
  • Simple map by address: 

    mapInit(address, map container ID, link to the pin image, user interface (true/false), offset (true/false) - optional);
example:
mapInitAddress("Fort Collins, Colorado 80523, USA","estate-map","images/pin-house.png", true);
  • Street view by latitiude & longitude: 

    streetViewInit(latitiude, longitiude, street view container ID, auto rotate panorama (optional));
example:
streetViewInit(39.6282,-86.1320,"estate-street-view", true);
  • Street view by Pano ID: 

    streetViewInitId(panoId, street view container ID, auto rotate panorama (optional));
example:
streetViewInitId("-AW3I6pOWMa4%2FVstMdShm9zI%2FAAAAAAAARI8%2FSckW-Xs5LJ0","estate-street-view", true);
  • Map with many pins and info boxes: 


    var locations = [
	[latitiude, longitiude, link to the pin image, infobox link, infobox image, infobox title, infobox price], //pin 1
	[latitiude, longitiude, link to the pin image, infobox link, infobox image, infobox title, infobox price], //pin 2
	...
	[latitiude, longitiude, link to the pin image, infobox link, infobox image, infobox title, infobox price], //pin n
];
offersMapInit(map container ID,locations);
example:
var locations = [
	[40.6128,-73.9976, "images/pin-house.png", "estate-details-right-sidebar.html", "images/infobox-offer1.jpg", "Fort Collins, Colorado 80523, USA", "$320 000"],
	[40.7222,-73.7903, "images/pin-commercial.png", "estate-details-right-sidebar.html", "images/infobox-offer2.jpg", "West Fourth Street, New York 10003, USA", "$350 000"],
	...
	[41.6926,-87.6021, "images/pin-house.png", "estate-details-right-sidebar.html", "images/infobox-offer5.jpg", "E. Elwood St. Phoenix, AZ 85034, USA", "$300 000"]
];
offersMapInit("offers-map",locations);
  • Map with geolocalization:


    This map doesn't need any javascript code but it has special html markup. Search localization input should have this ID and name: id="geocomplete" name="geocomplete" nad map container should have id="submit-property-map".

6.2 Sliders

I use Swiper plugin to create slideshows. You can find advanced configuration options on it's page if you want to change it's behaviour. Main slideshow structure looks like this:



<!-- Slider main container -->			
<div id="swiper1" class="swiper-container">
	<a href="#" class="slide-prev"></a>
	<a href="#" class="slide-next"></a>
	<!-- Additional required wrapper -->
	<div class="swiper-wrapper">
		<!-- Slides -->
		<div class="swiper-slide">
			<div class="slide-bg swiper-lazy" data-background="images/slides/1.jpg"></div>
			<!-- Preloader image -->
			<div class="swiper-lazy-preloader swiper-lazy-preloader-white"></div>
			<!-- Slide text/content -->
			...
		</div>
		<div class="swiper-slide">
			<div class="slide-bg swiper-lazy" data-background="images/slides/2.jpg"></div>
			<!-- Preloader image -->
			<div class="swiper-lazy-preloader swiper-lazy-preloader-white"></div>
			<!-- Slide text/content -->
			...
		</div>
		... more slides
	</div>
</div>

Main container should have id="swiper1" for Homepage 1 style slider, id="swiper2" form Homepage 2 & 3 style or id="swiper-gallery" for estate details style.

Navigation arrows should have slide-prev or slide-next class. Each slide is a div with class="swiper-slide"

Slider uses lazy loading feature. It means that image is loaded just before it will be shown, not on page load. It speeds up the page. Object with "lazy" image should have class="swiper-lazy" and will look like this:



For standard image: 
<img data-src="path/to/picture-1.jpg" class="swiper-lazy">
For background image 
<div data-background="path/to/picture-2.jpg" class="swiper-lazy">
	<div class="swiper-lazy-preloader"></div>
</div>

9. Typography

Please refer to this page code http://apartment-html.chart.civ.pl/typography-no-sidebar.html to check typography elements.

Template is built with Bootstrap, so all Bootstrap Helper Classes are working. I've also prepared some other mainly to have control of the margin:

  • Section paddings: Default section top and bottom padding is 90px. At those classes to change it: top-padding-45, bottom-padding-45, no-top-padding, no-bottom-padding, no-padding
  • Section shadow: You can add a top, bottom or both side shadow: section-top-shadow, section-bottom-shadow, section-both-shadow
  • To margins: Easily add vertical space with those classes: margin-top-15, margin-top-30, margin-top-45, margin-top-60, margin-top-75, margin-top-90, margin-top-105
  • Nagative margin: if you want to utilize auto paragraph top padding, you can add this class: negative-margin

9.1 HTML SELECT

Template uses Bootstrap Select to create nice multiselect prepacement for HTML SELECT.
You just create regular SELECT element and add "bootstrap-select" class to it. You can also add some more configuration options (multiple allows to select multipe fields and multiple data-actions-box="true" adds "select all/deselect all" buttons): 

<select name="country1" class="bootstrap-select" title="Country:" multiple data-actions-box="true">
		<option>United States</option>
		<option>Canada</option>
		<option>Mexico</option>
</select>

Some more options can be found here. Please note that "Live Search" option described on this page is not working in Internet Explorer 9 and it's why I didin't include it.

You can use onChange events to this select boxes just like to regular HTML tag. Here is the sample jQuery function: 

$( "select[name='country1']" ).change(function() {
	console.log( $( "select[name='country1']" ).val() ); //prints selected values in the console
});

10. RTL Pages

Template is adapted to work with RTL but some elements can require adjustments. Here are tow RTL pages you can use as a starting point: Homepage and Estate Details.

To convert page into RTL you must add those stylesheet to your HTML <head> section (after other styles):



<link rel="stylesheet" type="text/css" href="css/bootstrap-rtl.css">
<link rel="stylesheet" type="text/css" href="css/rtl.css"

You can change elements position by adding pull-left and pull-right classes or by changing columns order.

define('YOUR_EMAIL','mail@somedomain.tld');

If you use PHP mail() function, you don;t have to do anything else. If you use SMTP you need to add some more details in the same file. Ask your hosting provider if you don't know what to put below:



$smtpHost = ""; // SMTP servers
$smtpUsername = "";  // SMTP username 
$smtpPassword = ""; // SMTP password
$smtpFrom     = ""; // SMTP Sender email
$smtpFromname = ""; //Sender name

If you want to change error and success text you will find them on the same file.

If you don't want to use this script, delete following line from your html files (located at the end of head section):



<script type="text/javascript" src="js/validate.js"></script>

13. High Resolution (Retina) screens

Template is preaper to work with high resolution screens like Apple Retina ( https://en.wikipedia.org/wiki/Retina_Display ).

High resolution screens require bigger raster graphics, other way images can look a bit blurry. This is why all images have double size, and they are scalled down on regular screens. So to have nice high resolution photos just ave them with doubeled size (for eg. 400x200px instead of 200x100px).

Use higher compression of your photos to avoid large files and long loading time. Demo images are compressed with jpeg quality about 20%-40%. Compression artifacts are not visible because image in downsized twice on the website.
